passport.js

This file implements two passport strategies instrumental in authenticating requests to the Api endpoints. The local strategy is first used when a user logs in to use the app. Once the user is authenticated and a json web token iscreated, subsequent http requests receive the json web token as headers. This validates the request by decoding the Json Web Token returned to the user, then checking the user ID from the payload against the users collection in the database.
